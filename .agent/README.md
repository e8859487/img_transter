# Agent 配置說明

## 重要規則

### 📋 FEATURES.md 同步規則

**每次執行任何指令前：**
1. 必須先讀取 `FEATURES.md` 文件
2. 了解當前專案的所有功能

**每次執行指令後：**
1. 檢查是否有新增、修改或刪除功能
2. 如果有任何功能變更，必須更新 `FEATURES.md`
3. 保持文件內容與實際程式碼功能同步
4. 每個功能使用 2 行精簡句子描述

### 📝 更新原則

- **新增功能**：在對應分類下新增功能說明，或創建新分類
- **修改功能**：更新對應功能的描述文字
- **刪除功能**：移除對應的功能說明
- **重構代碼**：如果只是重構而功能不變，則不需更新

### 🎯 目標

確保 `FEATURES.md` 永遠是專案功能的最新、準確的文檔。
讓任何人都能快速了解這個工具提供哪些功能。
